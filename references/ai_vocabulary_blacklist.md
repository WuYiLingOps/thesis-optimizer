# AI高频毒词表及重构指南

> **返回**: [SKILL.md](../SKILL.md)

## 概述

本文档收录中文学术论文中AI生成文本的高频标记词，分为三个风险等级。
优化时须逐一排查，按重构指南进行替换或改写。

---

## 🔴 一级毒词（强标记，必须清除）

这类词汇在GPTZero等检测工具中权重极高，出现即大幅拉升AI检测率。

### 句首/段首强标记词

| 毒词 | 替换策略 |
|------|---------|
| 值得注意的是 | 删除或改为具体陈述，如"实验中观察到..." |
| 需要指出的是 | 直接陈述该内容，去掉引导语 |
| 不难发现 | 改为"可以看出"或直接给出结论 |
| 综上所述 | 仅在章节末尾保留一处，其余删除 |
| 总的来说 | 删除或改为"整体而言" |
| 毋庸置疑 | 删除，直接陈述结论 |
| 显而易见 | 删除，直接陈述结论 |
| 不言而喻 | 删除，直接陈述结论 |

### 贡献描述强标记词

| 毒词 | 替换策略 |
|------|---------|
| 创新性地提出 | 改为"提出"，去掉"创新性地" |
| 首次提出 | 保留"首次"但需有文献支撑，否则删除 |
| 开创性工作 | 删除，用具体贡献描述替代 |
| 填补了...空白 | 删除，改为具体的性能对比数据 |
| 具有重要意义 | 删除或改为具体的应用价值描述 |

### 过渡强标记词

| 毒词 | 替换策略 |
|------|---------|
| 在此基础上，进一步 | 改为"进而"或直接衔接下文 |
| 与此同时 | 改为"同期"或重组句子结构 |
| 有鉴于此 | 改为"为此"或"针对这一问题" |
| 基于上述分析 | 改为"据此"或直接引出结论 |

## 🟡 二级毒词（中等标记，建议替换）

这类词汇单独出现风险较低，但密集出现时会显著提升AI检测率。

### 笼统形容词

| 毒词 | 替换策略 |
|------|---------|
| 全面 | 用具体范围替代，如"覆盖X个数据集" |
| 系统 | 用具体方法替代，如"按照X流程" |
| 深入 | 删除或用具体分析内容替代 |
| 广泛 | 用具体数量替代，如"在9个基准上" |
| 有效 | 用具体指标替代，如"MSE降低X%" |
| 显著 | 用具体数值替代，如"提升了X个百分点" |
| 大幅 | 用具体数值替代 |
| 明显 | 用具体数值替代 |

### 套路化连接词（密集出现时处理）

| 毒词 | 替换策略 |
|------|---------|
| 此外 | 每段最多保留1处，其余改为句式重组 |
| 另外 | 同上 |
| 同时 | 改为并列句或分号连接 |
| 进一步 | 改为具体的递进关系描述 |
| 相应地 | 删除或改为具体的对应关系描述 |

### 方法描述套路词

| 毒词 | 替换策略 |
|------|---------|
| 旨在解决 | 改为"针对...问题" |
| 致力于 | 改为具体的技术手段描述 |
| 聚焦于 | 改为"专注于"或直接描述研究对象 |
| 以期 | 删除，直接陈述目标 |
| 从而 | 保留但控制频率，每段最多2处 |

---

## 🟢 三级毒词（低标记，注意频率控制）

这类词汇本身无问题，但在AI生成文本中出现频率异常高，需控制密度。

### 频率控制词

| 词汇 | 建议频率上限 |
|------|------------|
| 然而 | 每章不超过5处，与"但"、"不过"交替使用 |
| 因此 | 每页不超过2处 |
| 通过 | 每段不超过2处 |
| 实现 | 每段不超过2处 |
| 提出 | 全文控制在合理范围，避免每段都出现 |
| 验证 | 与"证明"、"表明"、"说明"交替使用 |
| 表明 | 与"显示"、"说明"、"证实"交替使用 |
| 有效性 | 与"性能"、"效果"、"能力"交替使用 |

## 重构操作指南

### 操作原则

1. **术语保护优先**：专业术语（如模型名称、算法名称、评估指标缩写）不在替换范围内
2. **句内操作**：所有替换限制在单句范围内，不跨句重组
3. **保留原意**：替换后语义必须与原文完全一致
4. **学术基调**：替换词必须保持客观、严谨、中立的学术风格

### 操作流程

```
Step 1: 全文搜索一级毒词 → 逐一清除或替换
Step 2: 统计二级毒词密度 → 密集区域进行替换
Step 3: 统计三级毒词频率 → 超出上限的进行多样化处理
Step 4: 复查替换后的句子 → 确保语义准确、语法正确
```

### 典型重构示例

**示例1：一级毒词清除**

❌ 原文：
```
值得注意的是，本文提出的方法在长序列场景下表现出显著优势。
```

✅ 重构后：
```
在长序列场景下，本文方法的优势尤为突出，MSE相比基线降低了X%。
```

---

**示例2：笼统形容词替换**

❌ 原文：
```
实验在多个数据集上进行了全面而系统的验证，结果显示方法有效。
```

✅ 重构后：
```
实验在9个公开数据集上展开，涵盖能源、交通、气象三类场景，
结果显示本文方法在7个数据集上取得最优MSE。
```

---

**示例3：连接词密度控制**

❌ 原文：
```
此外，该模块引入了注意力机制。同时，为了提升效率，我们采用了
稀疏化策略。另外，在训练阶段，我们使用了数据增强技术。
```

✅ 重构后：
```
该模块引入注意力机制，并通过稀疏化策略控制计算开销。
训练阶段配合数据增强以缓解过拟合。
```

## 检查清单

完成词汇清理后，使用以下清单验证：

- [ ] 全文无一级毒词
- [ ] 二级毒词密度正常（每段不超过2处）
- [ ] "然而"与其他转折词交替出现
- [ ] "验证/表明/显示"等同义词多样化使用
- [ ] 所有笼统形容词已替换为具体数值或范围
- [ ] 专业术语未被误替换
